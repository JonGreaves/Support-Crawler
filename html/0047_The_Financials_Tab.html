Adding Reserves Manually

To add reserves, click the ADD RESERVE button. This opens the Edit Reserve page.

Fields

  • Section - select the section to which the claim applies

  • Previous Outstanding - this is automatically updated when you save any changes

  • Amount - enter the amount in the Amount field, or use the Arrow-Up and Arrow-Down buttons to adjust the same in increments or decrements of Â£ 1,000.00.

  • Last Updated - this field is updated automatically on adding the reserve.

Click the ADD RESERVE button to secure any changes. Alternatively, click the BACK button to discard any changes.

Note: If you exceed your user account reserve limits , a Reserve Limit Exceeded message is displayed stating the limit for your account. You can still add the reserve, but it will need to be confirmed by a user with the rights necessary to approve the total reserve amount.

Click CONFIRM to secure any changes. Alternatively, click CANCEL to discard any changes. You are returned to the previous page.

Note the added CONFIRM button, enabling users with the appropriate privileges to confirm the new reserve.

Pending Confirmation

A 'Pending Confirmation' reserve is a reserve which has been added with a higher amount then the user's reserve limit.

If you confirm a reserve in excess of your limits

the status of your entry will be marked 'Pending Confirmation', and a CONFIRM button is added to your entry.

To proceed, you have the following options:

  • Ask a member of staff with the appropriate reserve limit and the "Confirm Reserve" permission to confirms the pending reserve. For details on Confirm Reserve permissions, see The Tasks Tab.

  • Request to have your own reserve limit raised sufficiently
    For details on reserve limits, see The Authority Reserve Groups Page.

The Payments tab enables you to manage any payments made related to the selected claim.

Adding Payments

To add payments, click the ADD PAYMENT button. This opens the Payment Details page.


Fields

In the Payment Details pane, complete the following:

  • Payee - Select Claimant or Supplier, and then select as relevant to the selected payee.

  • Payment Method - Select the payment method from the drop-down list

  • Payee Details - Enter the payee details relevant to the method of payment, these details may be prefilled from previous payments on this claim to the same claimant

  • Payment Reason - Select the payment reason from the drop-down list

  • Amount Gross (inc VAT if any) - This is a read-only field

  • VAT Amount (Non-Recoverable) - Enter the amount or use the Arrow-Up and Arrow-Down buttons to adjust the same in increments or decrements of Â£ 100.00.

  • Linked Documents - Select any linked documents from the drop-down list

In the Reserves To Pay pane, complete the following:

  • Reserve - Select the kind of reserve from the drop-down list

  • Outstanding - This is a read-only field

  • Paid to Date - This is a read-only field

  • New Payment - Enter the amount or use the Arrow-Up and Arrow-Down buttons to adjust the same in increments or decrements of Â£ 1,000.00

Tick the Close Reserve check box on the right to close the reserve after adding the payment.

Click the ADD RESERVE TO PAYMENT button to make the new payment. The following pane is appended.

You still have the option to EDIT or REMOVE the newly added payment.

  • Click BACK to discard any changes

  • Click SAVE FOR LATER to secure any changes but set the status to 'Pending'

  • Click ADD PAYMENT to process the payment now

If you have selected cheque payment, a Print button enables you to issue an authorized cheque. When clicked, a window opens to confirm that a correspondence has generated a document to send to the claimant. When that document is sent, a linked file is added to the payment, stating this is being generated (about half way down the page). To refresh the documents contents, close and re-open the payment. A cheque can only be printed once.

Click the button to return to the initial summary view of the middle pane

Authorising Payments

Each level of authorisation needs a new user to authorise. This means that a given payment would be added by User X, pre-authorised by User Y, and authorised by User Z.

To pre-authorise payments, click PRE-AUTH on the right.

To authorise pre-authorised payments, click AUTH on the right.

Note: You can only click CANCEL CHEQUE for cheque payments that have been authorised and for which none of the payment has been recovered.

Removing Payments

Click the REMOVE button to remove Payments which have not been pre-authorised. This opens a Payment Confirmation window.

Are you sure you wish to remove this payment? - select a reason from the 'Select Reason' drop-down list. This is mandatory. This list is configurable through the Configurable Lists page. See The Configurable Lists Page.

Additional Notes - Add further details regarding the payment removal. This is mandatory if you have selected Other from the 'Select Reason' drop-down list above.

Click YES to remove the payment. Alternatively, click NO to keep the payment in place.

Note:

  • Pre-authorised payments cannot be removed

  • You may not have sufficient rights to (pre-)authorise any payments

  • Removed payments are not actually removed from the Payments pane but the Status field is set to 'Removed' instead.

Viewing Payments

When an incident is not read-only, then payments which are cancelled, removed, rejected or authorised will be displayed with the View button.

Click the VIEW button on the right for that payment. This opens the Payment Details pane and the History pane.

Note:

  • When an incident is closed (indicated by a Read Only Mode banner at the top), then payments will also be read-only, and only the View button is displayed

  • When an incident is locked (indicated by a Read Only Mode banner at the top), then payments will also be read-only, and only the View button is displayed

To view payment details for unauthorised payments, click PRE-AUTH on the right.

To view payment details for pre-authorised payments, click AUTH on the right.

Marking a Recovery Payment as Received

To mark a given recovery payment as 'received', click the RECEIVED button on the right for that payment. This opens the Recovery Payments pane.

Click RECOVERY RECEIVED to save all changes. Alternatively, click BACK to discard any changes and return to the Recovery Payments tab.

Removing Recovery Payments

The recovery payment is deleted instantly. Deletion cannot be undone and any payment deleted in error must be re-added manually.

To delete recovery payments, click the REMOVE button for the relevant payment.

Viewing Recovery Payments

To view recovery payments, click the VIEW button for the relevant payment. This opens the Recovery Details page. Use the Arrow-Down button on the right to expand individual Reserve Detail items.

Click BACK to return to the Recovery Payments tab.
