Sending a Document to a Claimant
The documents you select here use templates which often contain merge fields and
free text
.
Merge Fields
Merge fields are codes that enable
Mobius Claims
to automatically populate the selected document with the appropriate data, such as the
claimants name
,
address
,
policy details
, and so on. If you select a document which contains with certain types of merge fields, then some new drop-down lists will enable you to select which entity you want the respective fields to be populated with.
Example:
If you are sending a document to a third-party driver involved in a collision, and you want to address them as 'Dear <driver name>', then the document could have a merge field for the driver name which, when selected, will display a drop-down list of all the third-party drivers entered into
Mobius Claims
during FNOL. The same concept applies to:
Multi-instance FNOL questions
Authorised or cancelled payments
Recoveries
Free text
Free text
Free-form text codes, such as 'Correspondence.FreeText.Generic', enable you to enter 'your own', unpredefined text into the selected document on merging the selected document.
If the selected document in the pop-up screen contains a free-form text code, then a rich text editor is displayed for each one, enabling you to enter your own text and insert graphics like you would in Microsoft Word.
If the document has more than one free-form text code, then an editor is available for every free-text code found. These are shown individually however, with forward and backward arrows to navigate between them.
